CS61B实验小结

进度大纲排版

  • 进度都是按Lab和Proj区分的，Josh老师讲的Lecture基本快进看，但有些还是有点内容的。下面的课程N指的是官网的LectureN，Lecture的总结也在下面了

Week1

Lab 1: JSetting Up Your Computer IntelliJ Setup

  • 安装Intellj IDEA、Java JDK、Git等工具
  • 同步CS61b Skeleton Spring 2021 Github 仓库，在google上就能够找到对应的地址。老师的代码框架
  • 强烈建议安装IntelliJ的Java Visualizer插件，调试可视化挺好用

HW 0: HW0: Basic Java Programs (之前学过就没做了)

  • 涉及到一些Java最基本的概念以及方法，没有学过Java的话，强烈推荐一定要去做

Project 0 2048

  • 有两部分，实现计分和移动，移动是通过只实现向上移动的操作加上旋转游戏盘面实现四个方向的移动的，有点难
  • 由于系统语言是中文的会出现上下左右失灵，需要系统语言改成英文或者修改proj0里控制上下左右移动的代码，改成WASD啥的

week 2

  • week 2 主要讲解如何使用Java的debugger，以及如何使用Junit来进行测试

Lab 2 Debugging

​ (这个实验有前置小实验指导书，得在官网CS61B官网找到指导书才行)

  • 这个Lab 2 的实验我整整配置了半天时间才弄好，下面是我我踩过的一些坑。
  • 在配置Lab 2 之前，根据提示，先进入Lab 2 setup 里面，跟着教程一步步做。
  • 因为git submodule update --init，这个命令是将cs61b的library下载到本地，因为cs61 b 的library是一个submodule，它链接到另外一个地方git仓库，所以要用这个命令来下载。
  • 如果命令无法下载submodule的话，我们可以直接去到submodule的链接里面下载，然后放到本地的cs61b的library里面。
  • 如果不用这个命令的话，就会出现java: package jh61b.junit does not exist的错误。
  • 只要完成了cs61b setup的设置，能够通过导入XML文件来导入cs61b的library，就可以正常运行Lab 2 了。
  • 下图为代码仓库中的submodule：
  • 在cs61b skeleton中会有这种标志的文件夹，在使用git clone的时候是不会下载的，这个是我们链接到别处仓库的。所以要使用git submodule update --init来下载。或者是手动下载，然后放到对应的文件夹里面。

Project 1 用链表和数组实现Deque & Guitar Hero

A部分 Deque

​ 用链表和数组实现Double-Ended-Queue(双端队列,)

​ 链表就用数据结构课上的Node节点创建前驱和后继来实现，使用的circle圈型实现头尾相接。主要是构造函数初始化一个起点(Start = head = end)起点的前驱指向后继，后继指向前驱，用size记录大小。之后的插入头部和尾部就在起点的右边和左边插入。

​ 数组方法有点抽象，有前后两个定位插入节点，从中间往两边扩容。

Week 3

Project 2 Gitlet实现

​ 耗时一周时间，翻了许多资料终于做完了。

  • Git的内部结构
    • blob(Binary Large Object): 文件的保存内容。一个文件可能对应多个blob：每个blob在不同的Commit中被跟踪。
    • TreeMap类: 映射文件名称(name)到blob的reference，或者是映射文件名称到其他tree(子目录)的引用。
    • commits: 日志消息，其他元数据(提交日期、作者等), 对树的引用和对Parent Commit的引用。repository还维护了分支头到提交引用的映射(以便某些重要的提交具有符号名称)

以下是实验要注意的几点：

  • 要把操作系统时间地区显示改成English(US)，要不然testing过不去
  • 先看给的设计文档最开始的几个介绍视频，把最开始大概要干什么搞清楚
  • 实验一开始要实现的Repository类的每个接口都错综复杂，需要提前设计好上面说的Git结构才能完成，并且越到后面需要联系前面的越多，所以要配合整体框架实现。
  • File类的创建文件名用英文就行，加下划线会出bug，Util.join函数打开文件夹没问题，可以用与存各类文件夹全局路径名
